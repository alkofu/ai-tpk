# Harness Feature Availability

This document lists which features are available per supported harness. Not all harness capabilities have equivalents in other harnesses — this is by design.

## Feature Matrix

| Feature            | Claude Code | OpenCode |
|--------------------|:-----------:|:--------:|
| Agent definitions  | Yes         | Yes      |
| Global instructions| Yes         | Yes      |
| Skills             | Yes         | No       |
| Hooks              | Yes         | No       |
| Slash commands     | Yes         | No       |
| Settings (plugins) | Yes         | No       |
| Agent modes        | No          | Yes      |
| Permission config  | No          | Yes      |

## Claude Code-Only Features

The following features live in `claude/` and are installed only when `--harness claude` (the default) is used. They have no OpenCode equivalents and are not generated or migrated by adapters.

### Skills (`claude/skills/`)

Skills are reusable instruction sets that Claude Code loads on demand. Two skills are mandated globally via `claude/CLAUDE.md`:

- **`commit-message-guide`** — enforces conventional commit format on all git commits
- **`open-pull-request`** — standardises PR creation across all projects

OpenCode users do not have access to these skills. Equivalent behavior must be configured directly in the OpenCode agent system prompt or global instructions (`~/.config/opencode/AGENTS.md`).

### Hooks (`claude/hooks/`)

Hooks run shell scripts at specific points in the Claude Code session lifecycle. Two hooks provide automatic session logging:

- **`talekeeper-capture.sh`** — SubagentStop hook; appends raw sub-agent events to `logs/talekeeper-raw.jsonl` after each sub-agent completes
- **`talekeeper-enrich.sh`** — async Stop hook; processes the raw log into a structured enriched JSONL chronicle at session end

These hooks are configured in `claude/settings.json` and are Claude Code-specific. OpenCode users will not have automatic session logging. The Talekeeper agent itself can still be invoked manually in OpenCode, but without the raw event log there is no input for it to narrate.

### Slash Commands (`claude/commands/`)

Slash commands provide quick workflow shortcuts within Claude Code (e.g., `/open-pr` to invoke the `open-pull-request` skill workflow). OpenCode does not support slash commands.

### Settings (`claude/settings.json`)

`settings.json` configures Claude Code-specific behaviour: plugin marketplaces, hook definitions, and other runtime settings. OpenCode uses a separate `opencode.json` global config file. No migration of `settings.json` content is performed.

### Global Instructions (`claude/CLAUDE.md`)

`CLAUDE.md` is loaded by Claude Code at session start for every project. It is hand-edited and is not generated by any adapter. The OpenCode equivalent is `~/.config/opencode/AGENTS.md`, which is generated by `adapters/to-opencode.sh` and installed to `~/.config/opencode/AGENTS.md`.

## OpenCode-Only Features

### Agent Modes (`mode:`)

OpenCode supports a `mode` field on each agent: `subagent`, `primary`, or `all`. This controls how the agent is invoked and what permissions it inherits. Claude Code has no equivalent; the field is stored in the `opencode:` extension block of each source agent and emitted by the OpenCode adapter.

### Permission Arrays (`permission:`)

OpenCode uses a `permission` array in agent frontmatter to control fine-grained tool access. Claude Code uses a `tools` string. These are stored separately in the `claude:` and `opencode:` extension blocks of each source agent file; adapters emit the correct format for each harness.

## Sub-Agent Orchestration Limitation

The DungeonMaster agent uses the `Agent` tool to delegate work to specialist sub-agents (Pathfinder, Bitsmith, Ruinor, etc.). The `Agent` tool is Claude Code-specific and has no equivalent in OpenCode. The generated `opencode/agents/dungeonmaster.md` includes a comment explaining this limitation. DungeonMaster will have significantly reduced orchestration capability in OpenCode.

## Tool Name Mapping

Claude Code uses capitalized tool names; OpenCode uses lowercase names under `permission`. The full mapping is documented in [`src/FORMAT.md`](/src/FORMAT.md#tool-name-mapping).
